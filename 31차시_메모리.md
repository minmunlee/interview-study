## ✏️ 메모리


<details>
  <summary>Redis 메모리가 초과되었을 때의 문제를 방지하기 위한 방안에 대해 설명해주세요.</summary> 
  데이터를 레디스의 메모리 용량보다 많이 사용하게 된다면 swap 메모리를 사용하게 됩니다. swap은 안정적이지만 디스크의 공간을 메모리처럼 사용하게 되어 속도가 매우 떨어져 성능 저하로 이어지는 문제가 발생할 수 있습니다. 

  이를 해결하기 위해 maxmemory 설정 방법이 있습니다. redis가 사용할 수 있는 메모리의 최대 양을 제한해서 메모리 부족으로 인한 장애를 방지할 수 있습니다. maxmemory-policy 옵션을 통해 메모리가 가득 찼을 때 어떤 데이터를 삭제할 지 설정합니다. 또는 TTL(데이터 만료 시간)을 설정해서 불필요한 데이터를 사전에 삭제하기 위해 키에 만료 시간을 설정하는 방안이 있습니다.
</details>

<details>
  <summary>객체가 생성되었을 때 자바 메모리 중 어떤 메모리에 영향을 미치는지 설명해주세요.</summary> 
  참조형 데이터 객체가 생성이 되었을 때 실제 데이터는 Heap 영역에 저장됩니다. new 키워드로 인스턴스를 생성할 때 Heap 영역에 객체가 저장되고 Stack 영역에 생성된 객체에 대한 주소 값이 저장됩니다. Heap 영역의 데이터들은 Stack 영역에 저장된 주소값들을 통해서만 접근이 가능합니다.
</details>

<details>
  <summary>메모리 누수가 무엇인지 설명해주세요.</summary> 
   메모리 누수는 프로그램이 더 이상 필요하지 않은 객체에 참조를 유지하여 해당 객체가 가비지 컬렉션에 의해 회수 되지 않고 메모리를 계속 차지히게 되는 상황입니다. 이후 사용한 메모리가 점점 줄어들면서 OOM(Out Of Memory) 오류를 발생 시키게 됩니다.
</details>

<details>
  <summary>스택 오버플로와 힙 오버플로의 차이에 대해 설명해주세요.</summary> 
  스택 오버플로는 함수의 재귀 호출 등으로 스택 영역의 할당량이 초과될 때 발생하며, 힙 오버플로는 힙 영역에 동적으로 할당된 메모리보다 더 많은 데이터를 저장하려고 할 때 발생하는 버퍼 오버플로의 한 종류입니다.
</details>

<details>
  <summary>메모리 관리 기법 중, 페이징 기법과 세그멘테이션 기법의 차이에 대해 설명해주세요.</summary> 

   페이징은 메모리 공간을 고정된 크기의 페이지 단위로 분할하는 기법이며, 세그멘테이션은 공간을 논리적 단위로 분할하는 기법입니다. 페이징 기법을 사용하면 논리 메모리가 물리 메모리에 적절히 배치되기 때문에 외부 단편화를 줄일 수 있지만 내부 단편화 문제가 발생할 수 있으며, 반대로 세그멘테이션은 내부 단편화는 줄일 수 있지만 외부 단편화 문제가 발생할 수 있습니다.
</details>

<details>
  <summary>캐시 메모리에 대해 설명해주세요.</summary> 
   캐시 메모리란 중앙 처리 장치와 주 기억 장치간의 데이터 전송 속도를 높이기 위해 사용되는 고속 기억 장치입니다. CPU가 자주 사용하는 데이터를 미리 복사해두어 메인 메모리까지 거쳐가는 시간 없이 빠르게 데이터를 처리할 수 있도록 돕기 때문에 성능 향상에 큰 도움을 줍니다.
</details>
