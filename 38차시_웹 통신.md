## ✏️ 웹 통신


<details>
  <summary>한 컴퓨터가 다른 컴퓨터로 데이터를 요청할 때의 일어나는 캡슐화와 비캡슐화 과정에 대해 설명해주세요.</summary> 
  한 컴퓨터가 다른 컴퓨터로 데이터를 요청할 때 보내는 값들이 캡슐화되어 다른 컴퓨터로 전달되고 데이터를 받은 컴퓨터는 비캡슐화를 통해 요청 데이터를 읽을 수 있습니다.

  이때 캡슐화 과정이란 상위 계층의 헤더와 데이터를 하위 계층의 데이터 부분에 포함시키고 하위 계층의 헤더를 삽입하는 과정이고, 비캡슐화 과정이란 하위 계층에서 상위 계층으로 가며 각 계층의 헤더 부분을 제거하는 과정을 의미합니다.
</details>

<details>
  <summary>SSL이 무엇인지 말해주세요.</summary> 
  SSL은 프로토콜의 하나로서 웹 서버와 브라우저 사이의 보안을 위해 만들어진 프로토콜 입니다.

  보안 세션을 기반으로 데이터를 암호화하며 보안 세션이 만들어질 때 인증 메커니즘, 키 교환 암호화 알고리즘, 해싱 알고리즘이 사용됩니다.
</details>

<details>
  <summary>HTTP 통신과 Socket 통신의 차이에 대해 설명해주세요.</summary> 
   HTTP 통신은 클라이언트의 요청이 있을 때만 서버가 응답하여 해당 정보를 전송하고 곧바로 연결을 종료하는 방식으로, 단방향적 통신으로 이루어져 있습니다. 요청을 보낼 때 내용을 기다리는 시간과 연결하는 시간이 걸리기 때문에 실시간성 연결보단 필요한 경우에만 서버로 접근하는 콘텐츠 위주의 데이터를 사용할 때 용이합니다.

  Socket 통신이란 서버와 클라이언트가 특정 포트를 통해 실시간으로 양방향 통신을 하는 방식입니다.

  HTTP 통신과 달리 포트를 통해 연결을 유지하고 있기 때문에 연결 지향형 통신이라고도 불립니다. 실시간 통신이 필요한 스트리밍 중계나 실시간 채팅과 같은 데이터를 즉각적으로 주고 받아야할 때 사용됩니다.
</details>

<details>
  <summary>웹의 동작 방식을 간단하게 설명해 주세요.</summary> 
  사용자가 브라우저에 웹 주소를 입력하거나 링크를 누르면  브라우저가 DNS 서버를 통해 도메인 이름에서 실제 주소인 IP주소를 찾습니다. 그 다음 브라우저가 해당 서버에 TCP를 이용해 HTTP 요청 메시지를 보내면 웹 서버가 이 요청을 처리하고 HTTP 응답 메시지를 보냅니다. 그러면 서버가 웹 사이트의 파일들을 패킷 단위로 브라우저로 보내면 브라우저가 그 파일을 해석해 웹 페이지를 구성하고 사용자에게 화면을 보여줍니다.
</details>

<details>
  <summary>HTTP 통신과 소켓 통신의 차이를 말씀해 주세요.</summary> 
   HTTP 통신은 요청과 응답의 비연결성 프로토콜로 이전 요청과 현재 요청이 독립적으로 처리되는 비상태성의 특징을 가지며, 소켓 통신은 반대로 클라이언트와 서버 간에 지속적인 연결을 유지하는 양방향 통신 프로토콜로 이전 상태와 현재 상태가 서로 영향을 미치는 상태성의 특징을 가집니다.
</details>

<details>
  <summary>HTTP 캐싱이란 무엇인지 설명해 주세요.</summary> 
   HTTP 캐싱은 웹 리소스를 사용자의 브라우저나 서버에 임시로 저장해두고 동일한 요청이 있을 때 빠르게 제공하는 기술로 Cache-Control 등의 HTTP 헤더를 사용해 캐시의 유효 기간, 재검증 여부 등을 설정해서 구현할 수 있습니다. 캐시된 리소스를 사용해서 서버로의 요청 수를 줄여 전체적인 웹 어플리케이션 성능을 향상시킬 수 있습니다.
</details>
